## Concerns

### Minor: hermes-agent install mechanism assumed
The task brief specified the binary lands at `~/.hermes/bin/hermes` after install.
The script achieves this via `pip install --prefix="$HOME/.hermes"`. If the actual
hermes-agent package uses a different install mechanism (e.g., a custom `install.sh`
bundled in the cloned source), this pip invocation may not produce the expected binary
path. The script degrades gracefully: it warns and continues if the binary is not found
after install rather than hard-failing.

### Minor: hermes-ctl.sh still references `docker exec` in configure.sh setup-crons.sh
The `setup-crons.sh` generated by `configure.sh` still uses `docker exec hermes-agent`
for cron management (out of scope for Task 4). This will need updating in a follow-on
task since hermes-agent now runs natively.

### Administrative: task-4-brief.md not found
The brief was not present at `.git/sdd/task-4-brief.md` or any searchable path.
The script was written from the requirements stated in the task prompt directly.
