# Native hermes-agent and Ollama Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move hermes-agent and Ollama from Docker containers to native host processes on Linux and macOS, while keeping all other services containerized and able to reach the native processes.

**Architecture:** Call official installers for both services. `ollama.ai/install.sh` on Linux creates a systemd service with automatic NVIDIA detection. `brew install ollama` on macOS uses Homebrew's launchd integration. `hermes-agent.nousresearch.com/install.sh --skip-setup` installs the Python+uv runtime on both OSes; on Linux `hermes gateway install` registers a systemd user unit; on macOS we write a launchd plist ourselves. Docker containers reach native processes via `host.docker.internal` with `extra_hosts: host-gateway` added to every service (required on Linux Engine; harmless on macOS Desktop).

**Tech Stack:** Bash, YAML (compose + litellm), XML (launchd plist), systemd user units, Homebrew (macOS)

## Global Constraints

- Target OS: Linux (systemd) and macOS (launchd/Homebrew). No other OSes.
- hermes-agent installer flag: `--skip-setup` skips the interactive API key wizard
- hermes-agent data: `~/.hermes/` (`HERMES_HOME` default, never overridden)
- hermes-agent binary after install: `~/.hermes/bin/hermes`
- hermes-agent API port: 8642
- Ollama port: 11434
- LiteLLM stays in Docker on port 4000; hermes reaches it at `http://localhost:4000/v1`
- Docker → host networking: `extra_hosts: ["host.docker.internal:host-gateway"]` on every service
- All shell scripts must pass `shellcheck -x` with zero errors
- Secrets: `$INSTALL_DIR/.env` (chmod 600, source of truth) and `~/.hermes/.env` (chmod 600, derived)
- No `docker exec hermes-agent` or `docker exec hermes-ollama` anywhere in scripts or docs after this change
